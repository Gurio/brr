# Command reference

This document describes the commands exposed by the `brr` CLI.  Each
command has a clear purpose and minimal options.  See `brr --help` for
up‑to‑date usage.

## `brr init`

Initialize the current Git repository for AI‑assisted operation.  The
process performs the following steps:

1. **Inspect:** scans for existing instruction files (`AGENTS.md`,
   `CLAUDE.md`, `GEMINI.md`, `.cursor/rules`), scripts, TODO files and other
   cues.
2. **Extract:** calls the configured executor in adoption mode to
   summarize the repository’s purpose, workflows and important commands.
3. **Propose:** generates a normalised set of files (`AGENTS.md`,
   `agent_state.md`, and tool‑specific shims as needed) reflecting the
   extracted intent.
4. **Present:** shows the proposed changes to the user and confirms before
   writing them to disk.
5. **Write:** writes the files and commits them if requested.

If the repository is not yet a Git repository, `brr init` will offer to
`git init` it.  It refuses to overwrite untracked files unless you move
them or commit them first.

## `brr run "<task>"`

Execute a single task through the configured executor.  The task is a
natural language instruction such as “fix failing tests in the auth
module”.  The `run` command resolves the repository’s configuration, starts
the executor with the prepared context and streams progress to the
terminal or chat.  At the end, it writes a summary to `agent_state.md`
and optionally commits the changes.

## `brr status`

Show a concise snapshot of the current project state:

- current mode (paused, incubating or live)
- selected executor
- current branch and clean/dirty state
- last run result (success/failure and summary)
- pending alerts or next steps (optional)

This command is optimised for quick checks.

## `brr report`

Produce a narrative report of what has happened recently in this
repository.  The report is suitable for reading on a phone and includes:

- current focus
- key decisions and discoveries
- next steps and open questions

The report is generated by the executor using the prompt in
`prompts/report.md`.

## `brr auth telegram`

Configure Telegram integration.  This command guides you through
entering your bot token and stores it in `.brr.local/telegram.json`.  It
verifies connectivity by sending a test message.  Telegram is optional;
you can always use the CLI without chat integration.

## `brr connect telegram`

Bind the current repository to a Telegram chat or topic.  After running
`auth`, this command lets you select a supergroup and a topic.  `brr` will
route messages related to this repo to the chosen topic.

## `brr up`

Start the brr daemon and register the current repository for managed
operation.  When a repo is “up”, the daemon routes incoming chat
messages, starts workers for tasks and schedules any periodic checks
defined in your configuration.  `brr up` is optional—`brr run` works
without it—but you need to call `up` at least once to enable chat
operations.
